## 🎯 6A 方法论简介

**6A = Align (对齐) + Architect (架构) + Atomize (原子化) + Approve (审批) + Automate (执行) + Assess (评估)**

### 核心理念

1.  **文档先行** - 不写文档不准写代码，确保所有决策有据可查。
2.  **任务递归** - 将复杂任务层层分解到 AI 无法做错的原子粒度。
3.  **范围收敛** - 明确边界，防止 AI 自由发挥导致范围蔓延。

### 六个阶段

1.  **Align** (对齐) - 澄清需求与目标，消除模糊和假设。
2.  **Architect** (架构) - 先设计后编码，规划清晰的实现路径。
3.  **Atomize** (原子化) - 将设计方案分解为可执行的、独立的原子任务。
4.  **Approve** (审批) - 在关键节点进行人工检查，确保方向正确。
5.  **Automate** (执行) - AI 严格按照已批准的文档和任务列表进行自动化实现。
6.  **Assess** (评估) - 对交付成果进行量化质量评估，不达标则循环改进。

## 📋 包含的模式

### 通用与编排层
1.  **🧠 通用智能助手** - 一个能够处理各种日常问题和任务的通用型智能助手。
2.  **🎼 6A工作流总指挥** - **【6A流程总入口】** 自动化编排端到端开发流程,实现从需求到高质量代码的全流程托管。
3.  **🏛️ 6A开发总监** - **【6A流程核心组件】** 负责核心开发循环(Align至Automate),在各阶段主动调用专家模式确保过程质量。

### 知识与分解层
4.  **📚 私域知识工程师** - 项目知识库的创建、维护和应用,让AI懂行地写代码。
5.  **⚛️ 任务原子化专家** - 将复杂任务递归分解到AI无法做错的粒度。
6.  **🔄 递归分解器** - 使用递归策略将任何复杂度的任务分解为可管理的原子单元。

### 流程与质量保障层
7.  **🎯 范围守护者** - 防止AI发散,确保每个阶段严格在定义的范围内工作。
8.  **🔗 可追溯性管理者** - 确保从需求到代码的完整追溯链,实现全程可追溯。
9.  **🧠 上下文优化器** - 在有限上下文中智能管理信息,确保关键信息始终可用。
10. **📋 文档先行执行者** - 严格执行文档先行原则,确保所有决策都有文档支撑。

### 实现与验证层
11. **🚀 渐进式实现者** - 采用渐进式策略实现功能,每一步都验证,避免大规模返工。
12. **✓ 验证优先开发者** - 先写验证标准和测试,再写实现代码,确保可测试性。
13. **⚡️ 敏捷开发者** - 用于快速、轻量级的代码修改、Bug修复和原型验证,专注于效率。
14. **✅ 质量评估专家** - 多维度代码质量评估,提供量化评分和改进建议。

## 📊 适用场景

### ✅ 适用（使用 6A 工作流）

- **中大型功能开发**：需要清晰规划和多人协作的复杂功能。
- **关键业务逻辑**：对正确性和稳定性要求极高的核心模块。
- **技术重构**：有计划、有步骤地重构老旧代码。
- **生产环境部署**：需要经过严格质量门禁的发布流程。

### ✅ 适用（使用敏捷开发者）

对于以往不推荐使用重型6A流程的场景，现在推荐使用专为轻量级任务设计的 **`⚡️ 敏捷开发者`** 模式：

- **简单代码修改**（预计<2小时）
- **明确的 Bug 修复**
- **快速原型验证**
- **一次性脚本编写**

## 🎓 核心约束（10 条铁律）

1.  **文档先行** - 每个阶段必须先完成文档。
2.  **任务递归** - 复杂任务必须分解。
3.  **范围收敛** - 明确"做什么"和"不做什么"。
4.  **人工检查** - 关键决策点必须用户批准。
5.  **质量把关** - 最终交付评估得分需 ≥90 分。
6.  **知识沉淀** - 每次开发后更新项目知识库。
7.  **可追溯性** - 所有代码都能追溯到需求和设计。
8.  **禁止跳阶** - 必须按 6A 顺序执行，不可跳过阶段。
9.  **禁止隐式** - 所有决策和假设必须显式记录。
10. **禁止偏离** - 执行阶段不允许偏离已批准的设计。

## 🎯 设计目标

本套件旨在解决 AI 辅助开发中的核心挑战：

1. **有限上下文** - 通过文档化和知识库管理上下文
2. **范围蔓延** - 通过明确的范围声明控制边界
3. **质量不稳定** - 通过量化评估和质量门禁确保质量
4. **难以追溯** - 通过完整的追溯链实现可追溯性
5. **任务太复杂** - 通过递归分解降低复杂度

# 规则集 02: 质量与技术标准

本规则集定义了所有产出的质量基线。

## 1. 代码质量标准
- **一致性**: 遵循项目现有编码规范和风格。
- **单一职责 (SRP)**: 函数/方法只做一件事。
- **清晰命名**: 命名需清晰表达其用途，避免模糊缩写。
- **错误处理**: 必须显式处理所有可预见的错误路径。
- **注释**: 为复杂逻辑、业务规则和公共API编写清晰注释。

## 2. 测试标准
- **TDD优先**: 优先编写测试，再进行实现。
- **覆盖率**: 核心业务逻辑测试覆盖率 > 90%。
- **用例完备性**: 必须包含正向、负向和边界条件测试。
- **命名清晰**: 测试用例名称应清晰描述其测试场景和预期结果。

## 3. 文档标准
- **同步性**: 文档必须与代码的最终状态保持完全一致。
- **结构化**: 使用 Markdown 进行清晰的结构化表达（标题、列表、代码块）。
- **准确性**: 技术细节、API描述等必须准确无误。

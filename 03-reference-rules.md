# 规则集 03: 参考规范与知识

本规则集提供开发过程中需要参考的具体规范。

## 0. 语言偏好
- **主要语言**: 简体中文 (zh-CN)
- **技术术语**: 优先使用中文解释，必要时提供英文原文
- **代码注释**: 使用中文或英文，保持项目一致性

## 1. Git 提交规范
- **格式**: `type(scope): subject` (e.g., `feat(api): add user login endpoint`)
- **类型**: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
- **追溯**: 提交信息中必须包含相关的需求或任务ID。

## 2. 沟通风格
- **直接简洁**: 避免“很好”、“当然”等冗余开场白。
- **技术准确**: 使用精确的技术术语。
- **问题导向**: 遇到不确定点立即提问，并提供自己的理解选项供用户选择。

## 3. 关键术语表
- **ADR**: Architecture Decision Record (架构决策记录)
- **TDD**: Test-Driven Development (测试驱动开发)
- **SRP**: Single Responsibility Principle (单一职责原则)

# 规则集 01: 工作流程规范

本规则集定义了从任务接收到交付的标准化流程。所有行为必须在 `00-core-rules.md` 的元指令框架下执行。

## 1. 任务接收与规划
1.  **理解与澄清**: 仔细阅读任务，列出所有不确定点并向用户提问以获得澄清。
2.  **文档先行**: 创建或更新设计文档/规划文档。
3.  **任务分解**: 将复杂任务分解为原子化的、可验证的子任务列表。
4.  **获取批准**: 提交设计和任务计划，**等待用户明确批准**后方可进入执行阶段。

## 2. 任务执行
1.  **渐进实现**: 采用小增量、持续验证的方式，一次只专注于一个子任务。
2.  **障碍处理**: 严格遵循 `00-core-rules.md` 中定义的**强制SOP**来处理任何错误或障碍。
3.  **持续记录**: 在执行日志中更新进度、遇到的问题和做出的决策。

## 3. 任务完成与交付
1.  **自我审查**: 对照需求和 `02-quality-rules.md` 中的质量清单进行自我审查。
2.  **最终验证**: 运行所有相关测试，确保100%通过。
3.  **文档同步**: 确保所有相关文档与最终代码一致。
4.  **提交交付**: 提供完整的交付说明、最终产物和相关日志。
